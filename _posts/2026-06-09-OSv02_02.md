# Operating System_02
***
   
## Monitor. 모니터
   
객체 지향 프로그래밍 언어에서 자주 사용되는 고수준 동기화 구조.
   
모니터 내부에는 한 번에 하나의 스레드만 진입해서 코드를 실행할 수 있도록 보장.
   
개발자가 직접 락을 관리하지 않아도 상호 배제(Mutual Exclusion)를 자동 제공
   
## Condition Variables. 조건 변수
   
스레드가 실행을 계속하기 전 특정 조건이 참인지 확인해야할 때 사용하는 큐.
   
스레드는 조건이 충족되지 않으면 이 변수에서 대기, 다른 스레드가 조건을 충족시킨 후 
신호를 보내면 깨어나는 방식.
   
## State Variable. 상태 변수
   
조건 변수와 함께 사용되는 명시적 변수(ex. fullEntries 정수).
   
세마포어와 달리 조건 변수 자체는 값을 가지지 않음. 
   
스레드가 대기해야 할지 여부를 결정하기 위해서 이 외부 상태 변수를 필수적으로 체크해야함.
   
## wait()
   
조건 변수에서 사용되는 연산.
   
호출한 스레드를 일시 중단(block)시키고 해당 조건 변수의 대기 큐에 넣음.
   
이때 스레드는 자신이 보유했던 모니터 내부의 락을 해제해서 다른 스레드가 들어올 수 있도록 함.
   
## signal()
   
조건 변수에서 대기 중인 스레드 중 하나를 깨워 실행 가능 상태로 만드는 연산.
   
깨어난 스레드는 다시 모니터 락을 흭득하기 위해 경쟁.
   
## Hoare semantics. 호어 방식
   
토니 호어가 제안한 방식. signal()이 발생하면 신호를 보낸 스레드가 즉시 중단되고, 
깨어난 스레드가 즉시 제어권을 넘겨받아 실행되는 방식.
   
이론적으로는 명쾌한데 실제 구현이 엄청 복잡함.
   
## notifyAll() / Broadcast. 브로드캐스트
   
자바의 notifyAll()처럼 특정 조건을 기다리는 모든 스레들르 한꺼번에 깨우는 연산.
   
어떤 스레드를 깨워야 할지 정확히 알 수 없는 경우(Converting Condition) 모든 대기자를 깨워 
각자 조건을 재확인하게 함으로써 시스템이 멈추는 문제를 방지.
