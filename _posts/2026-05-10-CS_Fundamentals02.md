# CS 기초 용어 정리_02
***
   
## 비트와 컨텍스트(Bits + Context)
   
시스템의 모든 정보(파일, 프로그램, 데이터 등)는 비트(0과 1)의 연속으로 
표현되며 이를 구분하는 방법은 해당 데이터 객체를 바라보는 문맥(Context)임
   
## 컴파일 시스템(Compilation System)
   
소스 프로그램을 시스템에서 실행 가능한 목적 파일로 변환하는 
과정으로 전처리기, 컴파일러, 어셈블러, 링커를 거침
   
## 버스(Buses)
   
시스템 내 컴포넌트들 사이에 정보를 전달하는 전기적 통로.
   
보통 워드(Word)단위로 데이터를 전송함.
   
## 프로세서(CPU)
   
메인 메모리에 저장된 명령어를 읽고 해석하는 엔진.
   
프로그렘 카운터(PC), 레지스터 파일, ALU로 구성됨.
   
## 캐시 메모리(Cache Memories)
   
프로세서와 메모리 간의 속도 차이를 줄이기 위해 SRAM 기술을 사용한 
작고 빠른 저장 장치(L1, L2, L3 캐시)
   
## 메모리 계층 구조(Memory Hierarchy)
   
상위 계층의 빠르고 비싼 저장 장치가 하위 계층의 느리고 
저렴한 장치의 캐시 역할을 하도록 구성된 구조
   
## 운영체제 추상화(OS Abstractions)
   
하드웨어를 관리하기 위한 핵심 추상화 개념.
   
프로세스(프로세서, 메모리, I/O의 추상화), 가상 메모리(메인 메모리, 디스크의 추상화), 
파일(I/O 장치의 추상화)이 있음.
   
## 암달의 법칙(Amdahl's Law)
   
시스템의 한 부분을 개선했을 때 얻을 수 있는 전체 성능 향상은 
그 부분이 전체 실행 시간에서 차지하는 비율에 달려 있다는 법칙
   
## 바이트(Byte)
   
8비트로 구성된 메모리의 최소 주소 단위
   
## 가상 주소 공간(Virtual Address Space)
   
기계어 프로그램이 메모리를 거대한 바이트 배열로 바라보는 개념적 이미지
   
## 16진수 표기법(Hexadecimal)
   
비트 배턴을 간결하게 표현하기 위해 사용하는 16진법 기반 표기
   
## 불 대수(Boolean Algebra)
   
0과 1의 값을 다루는 수학적 체계.
   
NOT(~), AND(&), OR(|), EXCLUSIVE-OR(^) 연산이 있음.
   
## 정수 인코딩
   
비부호형(Unsigned)과 음수를 표현하기 위한 2의 보수(Two's-Complement)방식이 있으며 
각각의 최대값(UMax, TMax)과 최소값(TMin)이 종요함.
   
## 부동 소수점(Floating Point)
   
IEEE 754 표준에 따른 실수 표현 방식으로 부호(s), 지수(exp), 소수부(frac) 세 필드로 구성.
   
## 반올림(Rounding)
   
실제 값을 부동 소수점 형식으로 정확히 표현할 수 없을 때 
가장 가까운 값으로 조정하는 방법(ex. Round-to-even)
   
## ISA(Instruction Set Architecture)
   
프로세서의 상태, 명령어 형식, 상태 변화를 정의하는 추상화 모델
   
## 레지스터(Registers)
   
CPU 내부의 고속 저장 공간.
   
x86-64에서는 %rax, %rsp(스택 포인터)등 16개의 64비트 레지스터를 사용함.
   
## 주소 지정 방식(Addressing Modes)
   
메모리 참조 시 유효 주소를 계산하는 방법
   
ex. Base + displacement, Scaled indexed
   
## 조건 코드(Condition Codes)
   
가장 최근 산술/논리 연산의 속성을 기록하는 단일 비트 레지스터(CF, ZF, SF, OF)
   
## 스택 프레임(Stack Frame)
   
함수 호출 시 인자 전달, 리턴 주소 저장, 지역 변수 할당을 위해 
할당되는 스택 영역
   
## 버퍼 오버플로우(Buffer Overflow)
   
입력 데이터가 할당된 공간을 초과하여 메모리를 덮어쓰는 보안 취약점
   
## HCL(Hardware Control Language)
   
하드웨어의 제어 로직을 기술하기 위해 사용하는 간단한 언어
   
## 조합 논리(Combinational Logic)
   
입력값에 의해서만 출력이 결정되는 논리 회로
   
ex. AND/OR Gate, MUX, ALU
   
## 파이프라이닝(Pipelining)
   
명령어 실행 과정을 여러 단계(Fetch, Decode, Execute, Memory, Write back)로 
나누어 중첩 실행함으로써 처리량(Throughput)을 높이는 기법
   
## 해저드(Hazards)
   
파이프라인의 다음 명령어가 다음 사이클에 실행되는 것을 
방해하는 상황으로 데이터 해저드와 제어 해저드가 있음.
   
## CPE(Cycles Per Element)
   
반복적인 연산을 수행하는 루프의 성능을 측정하는 척도
   
## 코드 이동(Code Motion)
   
반복되는 연산을 루프 밖으로 이동시켜 효율성을 높이는 최적화
   
## 루프 언롤링(Loop Unrolling)
   
루프의 반복 횟수를 줄이고 한 번의 반복에서 더 많은 요소를 처리하는 기법
   
## 메모리 에일리어싱(Memory Aliasing)
   
두 개의 포인터가 동일한 메모리 위치를 가리킬 수 있어 
컴파일러의 최적화를 방해하는 상황
   
## 명령어 수준 병렬성(Instruction-Level Parallelism)
   
현대 프로세서가 여러 명령어를 동시에 실행할 수 있는 능력
   
## 분기 예측(Branch Prediction)
   
조건부 분기문의 결과를 미리 추측하여 실행을 멈추지 않게 하는 기법
   
## SRAM, DRAM
   
전원이 켜져 있는 동안 데이터가 유지되는 빠른 SRAM(캐시용)
   
주기적으로 재생이 필요한 고밀도 DRAM(메인 메모리용)
   
## 지역성(Locality)
   
최근 참조한 위치를 다시 참조하는 시간적 지영성과 인접한 위치를 참조하는 공간적 지역성이 존재
   
## 캐시 히트/미스(Cache Hit/Miss)
   
요청한 데이터가 캐시에 있을 때를 히트, 없을 때를 미스라고 하며 미스 시 하위 계층에서 
데이터를 가져옴
   
## 캐시 조직(Cache Organization)
   
집합(S), 라인(E), 블록(B)로 구성, 직접 매핑, 집합 결합, 완전 결합 방식이 존재
   
## 메모리 산(Memory Mountain)
   
프로그램의 지역성에 따른 읽기 처리량을 3D 그래프로 시각화한 성능 특성
   
## 링커(Linker)
   
여러 코드와 데이터 조각을 모아 메모리에 로드되어 실행될 수 있는 하나의 파일로 결합하는 프로그램
   
## 재배치 가능 목적 파일(Relocatable Object File)
   
다른 목적 파일들과 결합하여 하나의 실행 파일을 만들 수 있는 형태의 이진 코드와 데이터를 포함함
   
## 실행 목적 파일(Executable Object File)
   
메모리에 직접 복사되어 즉시 실행될 수 있는 형태의 파일
   
## 심볼 해석(Symbol Resolution)
   
각 심볼 참조를 입력 목적 파일들에 정의된 정확히 하나의 심볼 정의와 연결하는 과정
   
## 재배치(Relocation)
   
개별 목적 파일의 코드와 데이터 섹션을 합치고, 각 심볼에 실행 시점의 메모리 주소를 할당하며 
모든 심볼 참조를 해당 주소로 수정하는 작업
   
## 정적 라이브러리(Static Library)
   
관련된 목적 모듈들을 하나로 묶은 파일로, 링커는 프로그램이 실제로 참조하는 모듈만 추출하여 실행 파일에 포함함
   
## 공유 라이브러리(Shared Library)
   
로드 타임이나 실행 시점에 동적으로 링크되어 여러 프로세스가 메모리 상의 단일 사본을 
공유할 수 있는 객체 파일
   
## 위치 독립 코드(PIC, Position-Independent Code)
   
메모리의 어느 주소에 로드되어도 수정 없이 실행될 수 있는 코드.
   
공유 라이브러리 구현의 핵심
   
## 예외(Exception)
   
프로세서 상태의 변화(이벤트)에 대응하여 제어 흐름이 갑자기 바뀌는 하드웨어와 
운영체제의 협력 메커니즘
   
## 인터럽트(Interrupt)
   
프로세서 외부의 I/O 장치로부터 온 신호에 의해 비동기적으로 발생하는 예외
   
## 트랩(Trap)
   
시스템 콜과 같이 의도적으로 발생하는 동기적 예외.
   
사용자 프로그램이 커널 서비스를 요청할 때 사용
   
## 폴트(Fault)
   
핸들러가 수정할 가능성이 있는 오류 조건.
   
성공 시 예외를 유발한 명령어를 재실행(ex. page Fault)
   
## 프로세스(Process)
   
실행 중인 프로그램의 인스턴스, 각 프로그램이 프로세서와 메모리를 독점한다는 
환상을 제공하는 핵심 추상화
   
## 문맥 전환(Context Swith)
   
커널이 현재 실행중인 프로세스의 상태(문맥)을 저장하고 미리 저장된 다른 프로세스의 
문맥을 복원하여 제어를 넘기는 과정
   
## 시그널(Signal)
   
어떤 이벤트가 발생했음을 프로세스에 알리는 작은 메세지.
   
소프트웨어적인 예외적 제어 흐름.
   
## 주소 번역(Address Translation)
   
가상 주소(VA)를 하드웨어(MMU)나 페이지 테이블을 사용하여 
실제 물리 주소(PA)로 변환하는 과정
   
## 페이지 테이블(Page Table)
   
가상 페이지를 물리 페이지로 매핑하는 정보를 담고 있는 메모리 내의 데이터 구조
   
## TBL(Translation Lookaside Buffer)
   
주소 번역 속도를 높이기 위해 MMU 내부에 설치된 페이지 테이블 엔트리 전용 캐시
   
## 메모리 매핑(Memory Mapping)
   
가상 메모리 영역을 디스크 상의 객체와 연관시키는 과정.
   
파일 공유나 프로그램 로딩의 효율성을 높임
   
## 동적 메모리 할당기(Dynamic Memory Allocator)
   
프로그램 실행 중에 힙 영역에서 가변 크기의 블록을 할당하고 해제하는 관리 프로그램(ex. malloc, free)
   
## 가비지 컬렉션(Garbage Collection)
   
더 이상 도달할 수 없는 메모리 블록을 자동으로 찾아 해제하는 암묵적 메모리 할당 방식
   
## 파일(File)
   
모든 I/O 장치를 바이트의 연속체로 모델링한 것.
   
시스템의 모든 입력과 출력은 파일을 읽고 쓰는 방식으로 수행
   
## 식별자(Descriptor)
   
파일을 열 때 커널이 반환하는 작은 정수.
   
프로세스가 파일에 접근할 때 사용하는 핸들 역할
   
## 쇼트 카운트(Short Counts)
   
네트워크 지연이나 EOF 도달 등의 이유로 요청한 바이트보다
적은 양의 데이터가 전송되는 현상
   
## Rio 패키지(Robust I/O)
   
쇼트 카운트를 자동으로 처리하여 안정적인 입출력을 보장하기 위해 개발된 래퍼 함수 묶음
   
## 표준 I/O(Standard I/O)
   
ANSI C가 제공하는 고수준 입출력 라이브러리(printf, scanf 등)로 버퍼링 기능을 포함하고 있음
   
## 동시성(Concurrency)
   
여러 개의 논리적인 흐름이 시간상으로 겹쳐서 실행되는 현상
   
## 쓰레드(Thread)
   
한 프로세스의 문맥 내에서 실행되는 논리적 흐름.
   
같은 주소 공간 내에서 코드와 데이터를 공유.
   
## 세마포어(Semaphore)
   
공유 자원에 대한 여러 쓰레드의 접근을 동기화하기 위해 사용하는 비음수 정수 변수
   
## 교착 상태(Deadlock)
   
여러 쓰레드가 서로 상대방이 가진 자원을 기다리며 더 이상 진행되지 못하는 상황
