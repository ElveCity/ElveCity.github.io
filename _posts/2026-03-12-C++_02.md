# 구조체에 관하여
***
   
C언어에서의 구조체는 데이터의 집합, 데이터를 보관하는 컨테이너의 역할만 수행하였습니다. 
   
C++로 와서는 생성자, 소멸자, 멤버 함수를 가질 수 있고 상속, 다형성을 지원합니다. 구조체는 
클래스와 동일한 기능을 지니지만 접근 제어자(Access Modifier)의 기본값이 구조체의 경우 public, 
클래스의 경우 private가 됩니다. 
   
## 메모리 정렬과 패딩(Memory Alignment, Padding)
   
CPU가 메모리에서 데이터를 읽어올 시 효율성을 위해 컴파일러가 구조체 멤버들 사이에 빈 공간(패딩 바이트) 
를 끼워 넣습니다. 
   
예시는 다음과 같습니다.
   
```C++
#include <iostream>

struct DefaultUser{
	char grade; // 1byte
	int score; // 4byte
	char initial; // 1byte
};

int main()
{
	std::cout << sizeof(DefaultUser) << '\n'; // 6이 아닌 12로 나옵니다.

	return 0;
}
```
   
패딩을 할 때는 구조체 내 가장 큰 기본 자료형 크기의 배수대로 맞춥니다. 
위의 경우에는 int가 가장 큰 자료형이기 때문에 4바이트 단위로 맞추어 각 char에 3바이트의 
패딩으로 빈 공간이 들어갑니다. 따라서 char(1 + 3(padding) = 4byte), int(4byte), char(1 + 3(padding) = 4byte)
로 각 4바이트씩 할당되며 상단의 DefaultUser 구조체에서는 총 12바이트의 메모리 할당이 일어납니다. 
   
패딩은 자료형 크기 순서에 따라서도 달라지게 됩니다. 위의 구조체에서 큰 것을 먼저, 작은 것을 뒤에 오는 식으로 
정렬하게 되는 경우 전체 크기가 작아질 수 있습니다. 
   
```
struct DefaultUser{
	int score; // 4byte;
	char grade; // 1byte;
	char initial; // 1byte;
};
```
   
이 경우 int의 4바이트 할당 후, 연이어 1바이트씩 할당, 남는 2바이트를 패딩하여 총 구조체의 크기는 8바이트가 됩니다. 
   
## 클래스와 역할 구분
   
- 구조체(Struct)
   
데이터 그 자체에 집중하는 형태입니다. 
복잡해지는 로직, 캡슐화(Encapsulation)가 요구되지 않고 순수한 데이터 묶음을 나타내려 할 때 
사용합니다. 이 경우 POD(Plain Old Data) 또는 값 객체(Value Object)라 합니다.
   
불변성(Immutability)을 지향하거나, 모든 멤버가 공개되어 외부에서 자유롭게 읽고 써도
시스템에 별 문제 없는 DTO(Data Transfer Object) 역할 수행 시에도 사용합니다. 
   
- 클래스(Class)
   
객체의 행동(Behavior), 책임에 집중하는 형태입니다. 
내부 상태를 보호(Private)하고, 공개된 함수(public)를 통해서만 외부와 소통함으로써 객체의 무결성을 유지합니다. 
   
구조체의 예시는 다음과 같습니다. 
   
```C++
#include <iostream>
#include <string>

struct PlayerConfig{
	std::string playerName;

	PlayerConfig(std::string name, int hp, float speed)
```
