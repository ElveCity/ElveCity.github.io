# Operating System_09
***
   
## Event Loop. 이벤트 루프
   
이벤트 기반 애플리케이션 핵심 루프.
   
시스템은 발생한 이벤트를 기다리다가, 이벤트가 감지되면 해당 이벤트를 처리할 수 있는 
서비스 루틴(이벤트 핸들러)을 호출하는 무한 루프 실행.
   
## select()
   
이벤트 루프 내에서 사용되는 핵심 시스템 호출.
   
여러 개의 파일 서술자(file descriptors)를 동시에 감시하며, 
어떤 서술자에 데이터가 도착했거나 쓰기가 가능한 상태인지 확인해서 준비된 
작업이 있을 때만 반환.
   
## Blocking Interfaces. 동기 인터페이스
   
요청한 작업이 모두 완료될 때까지 호출자에게 제어권을 넘겨주지 않고 멈춰있는 방식.
   
## Non-blocking Interfaces. 비동기 인터페이스
   
작업을 시작하자마자 즉시 반환되어, 실제 작업이 background에서 진행되는 동안 시스템이 다른 일을 
할 수 있게 하는 방식.
   
## Event-baesd Server. 이벤트 기반 서버
   
단일 스레드와 이벤트 루프를 사용해서 여러 클라이언트를 처리하는 서버.
   
작업 스케줄링을 세밀하게 제어할 수 있지만, 루프 내부에서 블로킹(Blocking) 호출을 하면 
서버 전체가 멈추므로 절대 하면 안댐
   
## Polling. 폴링
   
비동기 I/O 작업이 완료되었는지 확인하기 위해 주기적으로 시스템 상태(ex. aio_error())를 
체크하는 기법.
   
수백 개의 I/O가 진행중인 경우 이를 효율적으로 관리하는 것이 중요함.
   
## Continuation. 컨티뉴에이션
   
비동기 작업이 완료된 후 다음 단계를 진행하기 위해 필요한 정보를 데이터 구조(ex. hash table)에 
기록해 두는 기법.
   
작업 완료 시 이 구조체를 찾아 이전에 하던 일을 마무리함.
