# 가상화(virtualization) 관련 내용입니다.
***
   
## 제한된 직접 실행(Limited Direct Execution) 메커니즘
   
- 제한적 직접 실행(Limited Direct Excution, LDE)
   
운영체제는 물리적 CPU를 여러 작업 간 공유, 가상화합니다. 
   
가상화를 위해 프로세스를 하드웨어에서 직접 실행시키는 직접 실행 방식으로 작업을 진행하는데 
이를 통해 프로그램이 하드웨어 CPU에서 네이티브 속도로 실행되게 하며 가상화 구현 시 발생할 수 
있는 과도한 성능 오버헤드 문제를 해결합니다. 
   
성능 확보와 시스템 자원의 제어권 유지를 위해 프로세스의 권한 제한을 위한 제한적 장치를 
하드웨어의 도움을 받아 구성합니다. 
   
- 프로세서 실행 모드 분리(User mode, Kernel Mode)
   
시스템 제어권 확보와 관련하여 두 가지의 실행 모드로 분리, 존재합니다. 
   
사용자 모드(User Mode)는 일반 어플리케이션에서 I/O 요청, 특정 제한 명령어를 실행할 수 없도록 
설계되어 있습니다. 위반 시, 하드웨어에서 예외 발생, 운영체제가 해당 프로세스를 종료할 수 있게 구성됩니다. 
   
커널 모드(Kernel Mode)는 모든 특권 작업과 자원에 대한 완전한 접근 권한을 가집니다. 
   
- 보호된 제어 이전(Protexcted Control Transfer), 시스템 콜(System Call), 트랩(Trap)
   
사용자 프로세스가 I/O 요청과 같은 특권 작업을 수행해야 할 경우, 하드웨어가 제공하는 
시스템 콜(System Call) 인터페이스를 이용합니다. 프로그램이 특수한 트랩(Trap)명령어를 실행하면, 
하드웨어는 현재 레지스터 상태를 커널 스택에 저장하고 특권 수준을 커널 모드로 격상시킨 후 
운영체제 내부의 지정된 핸들러로 점프하게 됩니다. 
   
요청된 작업 완료 후 운영체제는 return-from-trap 명령어를 통해 사용자 모드로 복귀, 중단 시점으로부터 
실행을 재개합니다. 
   
- 트랩 테이블(Trap Table) 설정, 부팅 시 보안 구성
   
부팅 시 트랩 테이블(Trap Table)을 설정함으로써 운영체제는 시스템 콜이나 인터럽트 발생 시 
실행될 코드의 위치를 제어합니다. 
   
기기 부팅 시 운영체제는 특권 모드에서 하드웨어에 시스템 콜 핸들러나 인터럽트 핸들러의 주소를 
등록하고 하드웨어는 다음 재부팅 전까지 이 위치를 기억합니다. 
   
테이블 설정 자체도 특권 작업으로 분류됩니다. 따라서 사용자 프로그램이 임의의 커널 주소로 점프하는 것을 
원천적으로 차단합니다. 
   
- 타이머 인터럽트(Timer Interrupt)를 통한 비협조적 제어권의 복구
   
타이머 인터럽트(Timer Interrupt)는 프로세스가 시스템 콜을 호출하지 않고 무한루프 등 
CPU를 독점하는 상태가 발생하는 것을 방지하기 위해 구현되었습니다. 하드웨어 기반 구조에 해당합니다. 
   
부팅 시 운영체제가 주기적으로 타이머 장치가 인터럽트를 발생시키도록 설정합니다. 
인터럽트 발생 시 하드웨어는 현재 프로세스를 중단하고 운영체제의 핸들러를 강제실행합니다. 
   
이는 프로세스의 협력 여부와 관계없이 운영체제가 CPU의 주도권을 주기적으로 되찾아 다른 작업을 스케줄링 
할 수 있도록 보장하는 메커니즘입니다. 
   
- 문맥 교환(Context Switch), 저수준 구현 메커니즘
   
운영체제 스케줄러가 현재 실행 중인 프로세스 교체 결정 시 문맥 교환(Context Switch)가 발생합니다. 
   
운영체제는 문맥 교환 과정에서 현재 프로세스의 범용 레지스터, 프로그램 카운터(PC), 커널 스택 포인터 등을 
해당 프로세스의 구조체에 명시적으로 저장하며 새로 실행될 프로세스의 상태를 복원합니다. 
   
스택 포인터가 전환되며 시스템은 중단되었던 다른 프로세스의 문맥으로 복귀하게 됩니다.
