# ComputerNetwork
***
   
## 기초 용어
   
### 호스트(Hosts) 및 엔드 시스템(End Systems)
   
네트워크에 연결된 모든 컴퓨팅 장치, 인터넷의 가장자리에 위치하며 
애플리케이션 프로그램을 실행
   
### 패킷(Packets)
   
소스 시스템이 데이터를 보낼 때 일정한 크기로 쪼개어 헤더를 붙인 정보의 묶음
   
### 프로토콜(Protocol)
   
둘 이상의 통신 엔티티 간 교환되는 메세지의 형식, 순서, 그리고 메세지 
전송 및 수신 시 취해야 할 행동 정의 규칙
   
### 회선 교환(Circuit Switching)
   
통신 세션 동안 경로상의 자원(버퍼, 링크 전송 속도)이 
예약되는 방식으로 전용 연결 제공
   
### 지연(Delay)
   
패킷이 전송되는 과정에서 발생하는 시간 지체. 
   
노드 처리 지연, 큐잉 지연, 전송 지연, 전파 지연으로 구성
   
### 처리량(Throughput)
   
단위 시간당 소스에서 목적지로 전송되는 비트의 양
   
### 프로토콜 계층(Protocol Layers) 
   
복잡한 네트워크 시스템 관리를 위해서 계층화한 모델.
   
인터넷 스택은 애플리케이션, 전송, 네트워크, 링크, 물리 계층의 5개 층으로 구성
   
### 클라이언트-서버 구조(Client-Server Architecture)
   
항상 켜저 있는 서버가 다수의 클라이언트 요청에 응답하는 구조
   
### P2P 구조(Peer-to-Peer Architecture) 
   
전용 서버에 의존하지 않고 간헐적으로 연결된 호스트(피어)들 간에 직접 통신하는 구조
   
### 소켓(Socket)
   
호스트 내의 애플리케이션 계층과 전송 계층 사이의 인터페이스.
   
애플리케이션이 네트워크로 데이터를 보내고 받는 통로.
   
### HTTP(HyperText Transfer Protocol)
   
웹 브라우저와 서버가 메세지를 교환하기 위해 사용하는 애플리케이션 계층 프로토콜
   
### DNS(Domain Name System) 
   
사람이 읽기 쉬운 호스트 이름(ex. www.google.com)을 컴퓨터가 인식하는 IP 주소로 
변환해주는 분산 데이터베이스 서비스
   
### CDN(Content Distribution Networks) 
   
대규모 콘텐츠(ex. video)를 효율적으로 배포하기 위해 전 세계에 분산 배치된 서버 시스템
   
### 다중화(Multiplexing)와 역다중화(Demultiplexing)
   
여러 소켓의 데이터를 수집해서 하위 계층으로 보내거나 
도착한 세그먼트를 적절한 소켓으로 전달하는 과정
   
### UDP(User Datagram Protocol)
   
비연결형이며 비신뢰적인 데이터 전송을 제공하는 가벼운 프로토콜
   
### TCP(Transmission Control Protocol)
   
연결 지향형이며 신뢰적인 데이터 전송, 흐름 제어, 혼잡 제어를 제공하는 복잡한 프로토콜
   
### 신뢰적인 데이터 전송(Reliable Data Transfer) 
   
하위 계층이 비신뢰적이더라도 데이터가 손상되거나 
손실되지 않고 순서대로 전달되도록 보장하는 메커니즘
   
### 혼잡 제어(Congestion Control)
   
네트워크 내 패킷이 너무 많아져 발생하는 과부하를 방지하기 위해 
송신측의 전송 속도를 조절하는 기능
   
### 포워딩(Forwarding)
   
패킷이 라우터의 입력 링크에 도착했을 때 이를 적절한 출력 링크로 
이동시키는 라우터 내부의 로컬 동작
   
### 라우팅(Routing)
   
소스에서 목적지까지 패킷이 거쳐갈 전체 경로를 결정하는 네트워크 차원의 프로세스
   
### IPv4 및 IPv6
   
인터넷 프로토콜의 버전. IPv6는 주소 부족 문제를 해결하기 위해 
128비트 주소 체계를 도입하였음
   
### DHCP(Dynamic Host Configuration Protocol)
   
호스트에 IP 주소 및 네트워크 설정 정보를 동적으로 할당해주는 프로토콜
   
### NAT(Network Address Translation)
   
사설 네트워크 내의 여러 장치가 하나의 공인 IP 주소를 
공유하여 인터넷을 사용할 수 있게 해주는 기술
   
### 라우팅 알고리즘(Routing Algorithm)
   
라우터 간에 최소 비용 경로를 계산하는 알고리즘.
   
링크 상태(LS)와 거리 벡터(DV) 두 가지 방식이 존재
   
### OSPF(Open Shortest Path First)
   
단일 자율 시스템(AS)내부에서 사용되는 링크 상태 기반의 라우팅 프로토콜
   
### BGP(Border Gateway Protocol)
   
서로 다른 자율 시스템(AS)간에 경로 정보를 교환하며 
인터넷을 하나로 묶어주는 핵심 프로토콜
   
### SDN(Software-Defined Networking)
   
데이터 평면의 포워딩과 제어 평면의 라우팅 로직을 분리하여 
중앙 서버(컨트롤러)에서 관리하는 기술
   
### ICMP(Internet Control Message Protocol)
   
네트워크 내 오류 보고 및 관리를 위해 호스트와 라우터가 주고받는 
제어 메세지 프로토콜
   
### MAC 주소(Media Access Control Address)
   
링크 계층에서 노드를 식별하기 위해 사용하는 
물리적이고 영구적인 48비트 주소
   
### ARP(Address Resolution Protocol)
   
IP 주소를 기반으로 해당 호스트의 MAC 주소를 찾아주는 주소 변환 프로토콜
   
### 이더넷(Ethernet)
   
유선 LAN에서 가장 보편적으로 사용되는 기술 표준.
   
CSMA/CD방식을 통해 다중 접속을 관리.
   
### 링크 계층 스위치(Link-Layer Switches)
   
도착한 프레임의 MAC 주소를 학습하고 이를 바탕으로 특정 포트로 
프레임을 전달하는 장치
   
### VLAN(Virtual Local Area Network)
   
물리적인 배선과 상관없이 소프트웨어적으로 구성된 논리적 로컬 네트워크
   
### 무선 링크(Wireless Links)
   
전파를 사용하여 통신하며 경로 손실, 간섭, 다중 경로 전파 등으로 
인해 유선보다 비트 오류율(BER)이 높음
   
### 기지국(Base Station)
   
무선 호스트를 네트워크 인프라에 연결해주는 핵심 요소.
   
무선 LAN의 AP나 셀룰러의 셀 타워가 이에 해당함.
   
### 핸드오버(Handover)
   
이동 중인 단말기가 하나의 기지국 범위에서 다른 기지국의 범위로 
이동할 때 연결을 유지하는 과정
   
### CSMA/CA
   
무선 네트워크에서 충돌을 감지하기 어렵기 때문에 사용되는 
충돌 회피 기반 다중 접속 프로토콜.
   
### 4G LTE / 5G
   
고속 데이터 통신을 위해 설계된 현대적 이동 통신 표준
   
### 기밀성(Confidentiality)
   
송신자와 의도된 수신자만이 메세지 내용을 이해할 수 있도록 암호화하는 것.
   
### 무결성(Message Integrity)
   
전송 과정에서 메세지 내용이 임의로 수정되지 않았음을 보장하는 것.
   
### 인증(Authentication)
   
통신하는 상대방이 실제 주장하는 본인이 맞는지 확인하는 것.
   
### TLS(Transport Layer Security)
   
전송 계층에서 암호화와 인증을 제공하는 보안 프로토콜.
   
HTTPS의 기반.
   
### 방화벽(Firewall)
   
네트워크 경계에서 인가되지 않은 트래픽의 출입을 통제하는 
보안 소프트웨어 또는 하드웨어.
