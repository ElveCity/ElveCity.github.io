# 가상화(virtualization) 관련 내용입니다.
***
   
- 제한된 직접 실행(Limited Direct Excution, LDE)
   
직접 실행은 프로그램을 가상화된 환경이 아닌 실제 물리적 CPU 위에서 그대로 실행하여
네이티브 수준의 최고 속도를 얻어내는 방식입니다. 이 직접 실행에서 프로그램이 시스템 전체를
독점하거나 큰 문제를 발생치 않게 하도록 운영체제는 하드웨어의 지원을 받아 제한을 둡니다.
   
이러한 LDE 메커니즘은 성능 극대화와 시스템 제어권의 유지를 양립하게 합니다.
   
- 이중 실행 모드(User Mode, Kernel Mode)
   
사용자 모드와 커널 모드는 시스템의 보안 등급과 자원 접근 권한을 나누는 경계로 이용됩니다. 
일반 어플리케이션은 입출력이나 메모리 접근에 제약을 크게 두는 사용자 모드로 활동되며 
신뢰받는 운연체제만이 모든 하드웨어 자원을 직접적으로 다루는 것이 가능한 커널 모드로 활동하게 됩니다. 
이를 통하여 작은 실수, 악의적인 의도를 가진 프로그램이 하드웨어를 직접 장악하는 것을 
물리적으로 차단하는 효과를 얻습니다. 
   
- 시스템 콜과 트랩(System Call, Trap)
   
시스템 콜(System Call)은 사용자 프로그램이 스스로 할 수 없는 고권한의 작업(파일 읽기, 프로세스 생성 등)이 
필요할 때 운영체제에 요청하는 것입니다. 이 때 트랩(Trap)이라는 하드웨어 명령어가 실행됩니다. 
트랩은 현재 프로그램의 상태를 저장하고 권한을 높여 커널로 진입하게 해 주는 역할을 수행합니다. 
이를 통해서 프로그램은 복잡한 하드웨어 제어 방식을 알 필요 없이 운영체제가 가능한 작업들을 
수행할 수 있게 됩니다.
   
- 트랩 테이블과 핸들러(Trap Table, Handlers)
   
트랩 테이블은 부팅 시 운영체제가 하드웨어에게 미리 전달하는 지침의 역할을 합니다.
   
시스템에 어떤 예외 상황이나 서비스 요청이 들어오는 경우, 하드웨어가 커널 내부의 어느 지점으로 정확히 
점프해야 할지를 미리 약속해둡니다. 이를 통해 프로그램이 임의의 주소로 침번하여 시스템에 문제가 발생하는 것을 
사전에 방지할 수 있게 됩니다. 핸들러는 검증된 경로로서 운영체제의 기능을 실행하게 합니다.
   
- 타이머 인터럽트(Timer Interrupts)
   
타이머 인터럽트는 자원을 독점하려 하거나 무한 루프에 빠진 프로그램으로부터 CPU의 주도권을 
강제로 되찾아오기 위해 하드웨어가 운영체제에 특정 시간 간격마다 커널 권한을 부여함으로써 
시분할(Time Sharing)을 가능하도록 합니다.

- 문맥 교환 (Context Switch)
   
문맥 교환은 현재 실행중인 프로세스의 모든 내용, 상태(레지스터)를 프로세스 구조체에 저장하고 
다음에 실행될 프로세스의 정보를 CPU로 불러오는 작업입니다. 
   
이 과정에서 하드웨어적 비용이 소모되나 이 전환을 통해 수많은 프로그램이 동시에 작동하는 것처럼 
보일 수 있도록 합니다.
