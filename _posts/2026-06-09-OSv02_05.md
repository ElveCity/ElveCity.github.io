# Operating System_05
***
   
## Non-Deadlock Bugs. 비교착 상태 버그
   
교착 상태가 아닌 병행성 버그의 총칭.
   
대부분의 병행성 버그(약 97%)는 원자성 위반이나 순서 위반이라는 두 가지 유형에 속함.
   
## Atomicity Violation. 원자성 위반
   
여러 메모리 참조 간 예상된 원자적 실행이 보장되지 않아서 발생하는 버그.
   
예를 들어 어떤 변수가 NULL인지 확인 후에 접근하는 사이 다른 스레드가 
그 변수를 NULL로 만들어서 발생하는 충돌 등이 해당.
   
해결책으로는 해당 코드 영역을 Lock으로 감싸서 원자성을 보호하는게 일반적.
   
## Order Violation. 순서 위반
   
두 메모리 접근 간 기대한 순서가 뒤바뀌어 발생하는 버그.
   
스레드 A가 초기화 완료 후 스레드 B가 해당 값을 읽어야 하는데, 순서가 어긋나서 
초기화 전의 값을 읽으려 할 때 발생.
   
조건 변수나 세마포어를 써서 실행 순서를 강제로 해결 가능.
   
## Deadlock. 교착 상태
   
스레드들이 서로가 보유한 자원(락)을 기다리며 무한히 대기하는 상태
   
자원 간의 순환 대기(Circular Wait)가 발생 시 나타나며, 시스템 전체의 정지를 유발.
   
## Partial Ordering. 부분 순서화
   
교착 상태를 방지하기 위해 도입하는 기법.
   
시스템 내의 락들을 일정한 순서대로 흭득하도록 강제하는 것.
   
모든 락에 대해 엄격한 순서를 정하기 어려울 때, 락들을 그룹화해서 그룹 간의 흭득 순서를 정의, 
복잡한 시스템에서도 교착 상태 방지 가능.
   
## Lock-free / Wait-free synchronization.
   
명시적인 락을 사용하지 않고 자료구조를 안전하게 업데이트 하는 기법.
   
하드웨어가 제공하는 강력한 원자적 명령어인 Compare-and-swap 등을 활용해서 
여러 스레드가 동시에 접근해도 데이터 일관성을 유지하게 설계.
   
## Banker's Algorithm. 은행원 알고리즘
   
에츠허르 데이크스트라가 고안한 교착 상태 회피 알고리즘.
   
모든 작업이 필요로 하는 최대 자원량을 알고 있을 때, 자원 할당 시 교착 상태 발생 가능성을 
시뮬레이션해서 안전한 경우에만 할당 허용.
   
현실적으로 모든 작업의 자원 요구량을 미리 알기는 어려워서 임베디드같이 제한된 환경에서만 적용.
