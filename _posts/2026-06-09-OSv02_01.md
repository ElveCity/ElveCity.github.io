# Operating System
***
   
## gcc(GNU Compiler Collection)
   
컴파일 5단계
   
1) 전처리기(cpp)
   
#include , #define 과 같은 지시문을 처리, 확장된 소스 코드를 생성합니다.
   
2) 컴파일(cc1)
   
C 소스 코드를 하드웨어에 특화된 낮은 수준의 어셈블리 코드로 변환합니다.
   
3) 어셈블러(as)
   
어셈블리 코드를 기계가 이해할 수 있는 목적코드(object code)로 바꿉니다.
   
4) 링커(ld)
   
여러 목적 파일과 라이브러리를 하나로 묶어 최종 실행 파일을 생성합니다.
   
주요 플래그로는 모든 경고를 활성화화는 -Wall, 디버깅 정보를 포함하는 -g, 실행 파일 이름을 지정하는 -o, 
코드 최적화를 수행하는 -0 등이 있습니다.
   
## gdb(GNU Debugger)
   
- 디버깅 지원
   
프로그램 실행 중에 변수 값을 검사하거나 실행 흐름을 제어하여 논리적 버그를 찾는 도구입니다.
   
gdb를 효과적으로 사용하기 위해서는 컴파일 시 최적화(-0)를 끄고, 반드시 -g 플래그를 넣어서 
디버깅 정보를 포함해야합니다.
   
> 핵심 기능
   
- run
   
프로그램을 디버거 내에서 실행합니다.
   
- print
   
특정 시점의 변수 값을 확인합니다.
   
- breakpoint
   
특정 함수나 코드 줄에서 실행을 멈추게 합니다.
   
- next/step
   
코드를 한 줄씩 실행하며 논리적 흐름을 추적합니다.
   
## Makefile(make)
   
빌드 자동화로 대규모 프로젝트에서 복잡한 빌드 과정을 자동화하여 개발자의 생산성을 높여주는 도구입니다.
   
규칙(Rule) 기반 구조
   
- 타겟(target)
   
생성하고자 하는 파일 이름이나 실행할 동작(ex. clean)입니다.
   
- 필수 조건(prerequisite)
   
타겟을 만들기 위해 필요한 입력 파일들입니다.
   
- 명령어(command)
   
실제 빌드를 수행하는 작업이며, 행 시작 부분에 반드시 탭(tab)문자를 사용해야 합니다.
   
파일의 수정 시간(timestamp)을 비교하여 변경된 소스 파일과 관련된 부분만 선택적으로 재컴파일 하는 
증분 빌드가 있습니다.
   
## args / argv
   
- 명령행 인자
   
프로그램 실행 시 사용자가 입력한 추가 정보를 main() 함수로 전달하는 메커니즘 입니다.
   
- argc(Argument Count)
   
명령행에 전달된 인자의 총 개수를 나타내는 정수입니다.
   
- argv(Argument Vector)
   
각 인자 문자열을 가리키는 포인터 배열입니다.
   
argv은 항상 실행된 프로그램의 경로와 이름을 담고 있습니다.
   
배열의 마지막 요소(argv[argc])는 항상 NULL로 설정되어 배열의 끝을 알립니다.
   
프로그램 실행 시 운영체제가 이 값들을 적절히 설정해서 프로세스에 전달하게 됩니다.
